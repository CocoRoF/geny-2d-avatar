# 17. 리스크와 대응 (Risks & Mitigation)

> **한 줄 요약**: "이 제품이 왜 망할 수 있는가" 의 정직한 목록. 각 리스크에는 **조기 신호, 완화책, 플랜 B, 가드레일 지표**가 있다. 리스크는 정기적으로 리뷰되고 갱신된다.

---

## 1. 리스크 관리 원칙

1. **숨기지 않는다**. 가장 불편한 리스크일수록 먼저 정리.
2. **신호를 숫자로**. "위험하다" 는 감이 아니라 임계 지표로.
3. **플랜 B는 미리**. 사태가 터진 뒤 설계하지 않는다.
4. **리스크 오너가 있다**. 각 리스크마다 담당자와 리뷰 주기.
5. **연습한다**. 주요 실패 시나리오는 모의 훈련을 한다.

---

## 2. 리스크 분류

- **기술(Tech)**: 파이프라인·모델·인프라.
- **법률(Legal/Compliance)**: 저작권·프라이버시·규제.
- **시장(Market)**: 경쟁·수요·타이밍.
- **운영(Operational)**: 비용·공급자·인력.
- **평판(Reputation)**: 브랜드·커뮤니티.
- **안전/윤리(Safety/Ethics)**: 오용·편향·아동 안전.

## 4. 주요 리스크 상세

### R01 — AI 벤더 가격/정책 급변

- **시나리오**: nano-banana 호출 단가가 2배로 상승, 또는 특정 콘텐츠 생성 제한이 강화되어 우리 핵심 기능에 영향.
- **조기 신호**: 벤더 공지, 요금 인상 예고, API 에러 비율 급변, 콘텐츠 필터 차단 비율 증가.
- **완화**:
  - 어댑터 추상화(05 §2.3) 로 2–3개 벤더 병행 유지.
  - 장기 약정·볼륨 할인 협상.
  - self-host PoC (SDXL+ControlNet) 상시 준비.
  - 캐시/캐스케이드로 호출 절감.
- **플랜 B**: 2주 내 기본 벤더 전환 연습 분기 1회.
- **가드레일**: 벤더 1개 비중 ≤ 70%.

### R02 — AI 품질 미달

- **시나리오**: 얼굴 특정 위치 틀어짐이 잡히지 않아 반복 실패.
- **조기 신호**: AutoPass_rate 하락, Override_rate 상승, 사용자 이탈.
- **완화**:
  - 얼굴 특수 파이프라인(07 §6) 고도화.
  - 수동 앵커 편집 UI 조기 배포.
  - 실패 이슈 태그 상위 10 공격 주간 스프린트.
- **플랜 B**: "얼굴 재생성 대신 사용자 원본 보존 + AI는 주변만" 모드 기본화.
- **가드레일**: 얼굴 ValScore 평균 ≥ 72 유지.

### R03 — 저작권·IP 분쟁

- **시나리오**: 특정 작가 스타일과 유사한 스타일이 유행 → 법적 분쟁.
- **조기 신호**: 신고 접수 증가, 특정 스타일 프로파일 공유 급증.
- **완화**:
  - 스타일 등록제(16 §4.2) 운영.
  - 공개 공유 전 유사도 검사.
  - 약관상 사용자 책임 명시 + 내부 심사.
- **플랜 B**: 문제 스타일 신속 차단 + 소명 절차.
- **가드레일**: DMCA 접수 24h 내 초기 대응율 ≥ 90%.

### R04 — 경쟁 제품 출시

- **시나리오**: 대형 플랫폼 또는 Cubism 공식이 유사 기능 론칭.
- **조기 신호**: 업계 공지, 유사 베타 프로그램.
- **완화**:
  - 후처리·검수·커뮤니티 해자 강화.
  - 크리에이터 파트너십, 얼리 어답터 인센티브.
  - 핵심 차별점(반자동, 일관성) 을 지속 홍보.
- **플랜 B**: 가격·기능 차별화 패키지 신속 제시.
- **가드레일**: 브랜드 인지도 조사 분기 1회.

### R05 — 딥페이크/비동의 생성

- **시나리오**: 사용자가 타인 얼굴로 아바타를 만들어 피해 유발.
- **조기 신호**: 신고 급증, 소셜 미디어 이슈.
- **완화**:
  - 공인 얼굴 유사도 탐지 → 동의 확인 단계.
  - 미성년/공인 콘텐츠 강한 필터.
  - C2PA 호환 증명서 기본 탑재.
- **플랜 B**: 관련 기능 긴급 축소(신규 생성 일시 중단).
- **가드레일**: 월간 악용 신고 처리율 100%.

### R06 — Cubism 라이선스 변경

- **시나리오**: 상용 SDK 라이선스 인상·제한.
- **완화**: 자체 경량 런타임(11 §4) 로 부분 대체 가능 상태 유지.

### R07 — 비용 폭증

- **시나리오**: 사용량 급증 + 벤더 가격 상승 + 캐싱 적중률 저조.
- **조기 신호**: 아바타당 AI 비용 상승, 캐시 적중률 하락.
- **완화**:
  - 사용량 기반 과금 확장 + 쿼터.
  - 저비용 경로 자동 라우팅.
  - 배치 가격 유도.
- **가드레일**: 아바타당 평균 비용 ≤ $0.60.

### R08 — 핵심 인력 이탈

- **완화**:
  - 버스 팩터 관리: 각 영역 2명 이상.
  - 문서화 문화(이 docs/ 가 본보기).
  - 장기 보상 구조.

### R09 — 가용성 장애/데이터 손실

- **완화**:
  - Multi-AZ, PITR, 버전 관리 스토리지.
  - 복구 연습 분기 1회.
- **가드레일**: SLO 에러 버짓 모니터링.

### R10 — 규제 변화

- **완화**:
  - 모델 출력에 증명서(C2PA) 기본 옵션.
  - 지역별 약관/정책 지속 업데이트.
- **가드레일**: 규제 변화 감지 → 30일 내 적응.

### R11 — 프롬프트 인젝션·보안

- **시나리오**: 사용자가 악성 프롬프트로 필터 우회, 시스템 프롬프트 노출 시도, 또는 외부 파일에 프롬프트 인젝션.
- **완화**:
  - 프롬프트 격리: 시스템 프롬프트는 사용자 입력과 다른 채널.
  - 다층 필터.
  - 파이프라인 API 에 최소 권한.
- **가드레일**: 보안 사고 P1 0 유지.

### R12 — 시장 타이밍 오판

- **시나리오**: 제품은 훌륭한데 시장이 아직 준비 안 됨.
- **완화**: 사용자 조사·피봇 가능 범위 미리 정의.

### R13 — 엔터프라이즈 과잉 요구

- **시나리오**: 계약마다 다른 요구가 제품 복잡도를 폭증.
- **완화**:
  - 사전 정의된 엔터프라이즈 옵션 메뉴.
  - 커스터마이즈는 별도 비용 항목.

### R14 — 마켓플레이스 유해 콘텐츠

- **완화**:
  - 심사(업로드/거래) + 사용자 리뷰.
  - 신속 반응 채널.

### R15 — 데이터 주권/지역

- **완화**:
  - 지역 선택 가능, 데이터 잔류 약속.
  - 엔터프라이즈 전용 리전 옵션.

## 6. 블랙 스완 시나리오 & 대응 플레이북

### 6.1 "벤더 A 24h 장애"

- 즉시 벤더 B/C 로 라우팅, 대기열 우선순위 저하.
- 공지 페이지 업데이트.
- 사용자 환불/크레딧 크레딧 자동 부여.

### 6.2 "유명 IP 기반 스타일 바이럴"

- 노출 중단, 해당 스타일 공개 제한.
- 법무 팀 합류.
- 커뮤니케이션: "심사 중" 투명 공지.

### 6.3 "규제 공지: 생성형 AI에 워터마크 의무"

- 내부 기본 on, 기존 자산은 소급 라벨링 job.
- UI 안내.

### 6.4 "비용 폭증 예산 +40%"

- 가격/쿼터 조정 공지, 프리미엄 모델 일시 제한.
- 무료 티어 축소(기간 한정).

### 6.5 "주요 데이터 유출"

- 법정 통보 기한(72h) 준수.
- 크레딧/보상 프로그램.
- 포스트모템 공개.

## 7. 리스크가 낮다고 넘어가는 항목에 대한 메모

- R06(Cubism 변경), R08(인력 이탈), R15(지역) 는 낮지만 **발생 시 복구 긴 시간**. 연 1회 시뮬레이션.

---

## 8. 리스크 예산(Risk Budget)

- 에러 예산: 99.9% 가용성 월 ≈ 43분.
- 비용 예산: AI 비용 월 $15k (β) → 초과 시 자동 라우팅 변경.
- 신뢰 예산: P1 장애 월 1건 이하. 초과 시 동결 코드 주.

---

## 9. 변경 관리

- 큰 아키텍처 결정은 ADR 로 남긴다(02 §14).
- 정책 변경은 퍼블릭 체인지로그.
- 사용자 영향 변경은 30일 예고(필수 제외).

---

## 10. 열린 질문

- 엔터프라이즈 전용 리전 도입 시점.
- 규제 대응을 위한 내부 팀(Trust & Safety) 분리 시점.
- 버그 바운티 오픈 범위와 보상 한도.
