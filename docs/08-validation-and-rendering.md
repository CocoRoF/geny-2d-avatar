# 08. 검수 & 렌더링 (Validation & Rendering)

> **한 줄 요약**: **정적 이미지 검수는 거짓말을 한다.** 아바타는 움직일 때 평가해야 한다. 이 문서는 "표준 테스트 포즈 세트를 자동 렌더 → 지표로 채점 → 게이트 통과" 로 이어지는 검수기를 정의한다.

---

## 1. 원칙

1. **"움직일 때" 를 평가한다**: 10 포즈 이상 렌더 후 채점.
2. **다중 지표의 가중 합**: 한 지표로 판정하지 않는다.
3. **사람이 최종 판정한다**: 자동 점수는 "게이트" 지, "결재" 는 아니다.
4. **재현 가능**: 같은 아바타 + 같은 검수기 버전 = 같은 점수.
5. **설명 가능**: 점수와 함께 **무엇이 왜 깎였는지** 를 사용자에게 보여준다.

---

## 2. 검수기의 구성

```
             ┌──────────────────────────────┐
             │   Validation Render Service  │
             │                              │
  Avatar ──▶ │  1. Pose Executor            │ ──▶  PNG 시퀀스 / 썸네일
             │  2. Renderer (headless)      │ ──▶  짧은 MP4/WebM
             │  3. Metric Runner            │ ──▶  지표 JSON
             │  4. Scorer                   │ ──▶  score (0–100) + tags
             │  5. Gate & Route             │ ──▶  pass / review / reject
             └──────────────────────────────┘
```

각 단계는 버전을 갖고, 결과 JSON 에 기록.

## 3. 표준 포즈 세트 (Standard Test Pose Set)

### 3.1 공통 포즈 (모든 템플릿)

| Pose ID | 설명 | 파라미터 설정 |
|---|---|---|
| `neutral` | 기본 | 모두 기본값 |
| `blink_both` | 양쪽 눈 깜빡임 | `eye_open_l=0, eye_open_r=0` |
| `blink_left` | 왼쪽만 | `eye_open_l=0` |
| `smile_big` | 큰 미소 | `mouth_form=1, eye_smile_*=1` |
| `mouth_open` | 입 크게 | `mouth_open=1` |
| `mouth_a` / `mouth_i` / `mouth_u` / `mouth_e` / `mouth_o` | 모음별 | 각 vowel=1 |
| `head_turn_l30` | 왼쪽 30° | `head_angle_x=-30` |
| `head_turn_r30` | 오른쪽 30° | `head_angle_x=30` |
| `head_nod_up` | 위로 | `head_angle_y=20` |
| `head_nod_down` | 아래로 | `head_angle_y=-20` |
| `head_tilt_l` | 갸웃 왼쪽 | `head_angle_z=-20` |
| `head_tilt_r` | 갸웃 오른쪽 | `head_angle_z=20` |
| `breath_peak` | 호흡 정점 | `body_breath=1` |
| `eyes_look_up_r` | 시선 우상 | `eye_ball_x=1, eye_ball_y=1` |
| `brows_up` | 눈썹 올림 | `brow_l_y=1, brow_r_y=1` |
| `brows_frown` | 찌푸림 | `brow_l_angle=-1, brow_r_angle=1` |

### 3.2 템플릿별 확장 포즈

- **Full Body**: `leg_step_l`, `leg_step_r`, `skirt_sway_peak`.
- **Feline**: `ear_flick_l`, `tail_sway_peak`.

### 3.3 동영상 포즈 (Motion-based)

- `motion.idle_3s` — 정지 같지만 호흡/미세 흔들림.
- `motion.lipsync_hello_3s` — 립싱크 데모 문장.
- `motion.swing_head_4s` — 고개 좌우 풀 스윙.
- `motion.combo_3s` — idle + blink + lipsync 결합.

영상 기반 검수는 프레임별 지표를 **시계열 통계** 로 집계.

## 4. 렌더러 (Headless Renderer)

### 4.1 요구사항

- Cubism Web SDK 또는 자체 경량 2D 메쉬 렌더러를 **Headless WebGL / offscreen canvas** 로 구동.
- 결정론적 (frame 당 time step 고정).
- 속도: 1080p @ 60fps, 실시간 이상.

### 4.2 출력 포맷

- **PNG 시퀀스**: 포즈별 대표 프레임 1–3장.
- **썸네일 콘택트 시트**: 1 파일에 포즈 N개 그리드.
- **짧은 영상 (MP4/WebM, H.264/VP9)**: 2–4초.
- **디버그 오버레이**: 앵커/메쉬/마스크를 겹쳐 그린 버전 (옵션).

### 4.3 해상도

- 자동 검수: 512 해상도(빠름).
- 사용자에게 보여주는 프리뷰: 1024.
- 엔터프라이즈 QA 덤프: 2048.

## 5. 지표 (Metrics)

### 5.1 기하/구조 지표

| 지표 | 정의 | 합격 기준 |
|---|---|---|
| 자가교차 픽셀 수 | 메쉬 삼각형 뒤집힘으로 인한 픽셀 수 | 0 |
| 얼굴 뚫림 면적 | 고개 회전 시 face_base 가 hair_side 뒤로 노출된 면적 | ≤ 0.5% of face bbox |
| 파츠 이탈 | 파츠가 UV 박스 외부로 이동한 영역 | 0 |
| 경계 알파 급변 | 파츠 경계 2px 링의 알파 표준편차 | ≤ σ_th |
| 앵커 오차 (기대 vs 실제) | 각 포즈에서 이상 이동 | ≤ 3 px @ 2048 |

### 5.2 색/광원 지표

| 지표 | 정의 | 합격 기준 |
|---|---|---|
| context 내 ΔE_mean | 같은 color_context 파츠 평균색 편차 | ≤ 5 |
| 팔레트 이탈 | 지배색 vs 팔레트 최근접 | ΔE ≤ 12 |
| 광원 방향 오차 | 파츠별 광원 추정 vs 전역 | ≤ 15° |
| 하이라이트 일관성 | 하이라이트 방향 표준편차 | ≤ 12° |

### 5.3 "살아있음" 지표 (Liveness)

| 지표 | 정의 | 합격 기준 |
|---|---|---|
| blink 자연성 | `eye_open ∈ {0,0.5,1}` 의 면적-속도 곡선이 S-curve | 시각 일치 |
| lipsync 구분도 | 5 모음 렌더를 별도 분류기가 구분 | top-1 정확도 ≥ 85% |
| 스윙 복원 시간 | head 30° → 0° 후 head_side 정지까지 | 1.0–1.8s |
| 흔들림 점프 | 연속 프레임 차분의 표준편차 | ≤ σ_motion |

### 5.4 스타일 일관성 지표 (Semantic)

| 지표 | 정의 | 합격 기준 |
|---|---|---|
| CLIP 임베딩 거리(aavatar ↔ 스타일 프로파일) | 전체 썸네일 임베딩 평균과 프로파일 임베딩 코사인 거리 | ≤ d_th |
| 파츠 간 스타일 거리 분산 | 각 파츠 CLIP 거리의 표준편차 | ≤ σ_style |
| 정체성 유사도 (원본 대비) | 얼굴 FaceID 유사도 | ≥ 0.6 (원본이 있을 때) |

### 5.5 기술적 지표

| 지표 | 정의 | 합격 기준 |
|---|---|---|
| 렌더 성공률 | 모든 포즈 프레임이 NaN/에러 없이 렌더 | 100% |
| 렌더 시간 | 포즈 전체 렌더 시간 | ≤ 30s |
| 최종 PNG 무효 픽셀 | 완전 투명 전체 프레임 발생 | 0 |

## 6. 스코어링 (Scoring)

### 6.1 가중치

카테고리별 가중치는 템플릿과 사용자 목적에 따라 다르게.

기본(β) 가중:
- 기하/구조: 35%
- 색/광원: 25%
- Liveness: 25%
- 스타일: 15%

### 6.2 점수 계산

각 지표는 "합격 여부" 가 아니라 **연속 점수** 를 만든다:

```
score_i = clamp( 1 - (observed - pass_th) / (fail_th - pass_th), 0, 1 )
```

카테고리 점수 = 지표 점수의 가중 평균, 전체 점수 = 카테고리 가중 합 × 100.

### 6.3 단일 실패 오버라이드

특정 지표는 **단독으로 탈락** 시킨다:
- 자가교차 픽셀 > 0 → 점수 상한 60.
- 렌더 에러 → 자동 실패.
- 정체성 유사도 < 0.4 (원본 있는 경우) → 자동 실패.

## 7. 게이트 (Gate)

### 7.1 기본 임계값

| 점수 | 처리 |
|---|---|
| ≥ 85 | Auto Pass, 사용자에게 바로 프리뷰 |
| 78–84 | Pass + "작은 경고" 배지 |
| 65–77 | Needs Review — 인간 리뷰 큐로 |
| 50–64 | Auto Fix Suggested — 사용자에게 재생성 제안 |
| < 50 | Reject — 파이프라인 에러 로깅, 해당 파츠 재생성 자동 트리거 |

### 7.2 사용자 등급별 조정

- 무료: 기본값.
- 유료: 78 이상에서 Auto Pass (더 관대).
- 엔터프라이즈: 85 Auto Pass, 78 미만은 모두 인간 리뷰 (엄격).

### 7.3 오버라이드

- 사용자는 "Auto Fix Suggested" 라도 **그대로 사용** 을 선택할 수 있다.
- 이 경우 감사 로그에 `override: user_forced` 기록.
- Export 시에는 **경고 배너** 포함.

## 8. 사용자에게 보여주는 결과 패널 (Visible Report)

### 8.1 레이아웃

```
 ┌──────────────────────────────────────────────────────┐
 │ Validation Report              Score: 81 / 100  🟢   │
 ├──────────────────────────────────────────────────────┤
 │ [Geometry  ▸ 88]  [Color  ▸ 76]  [Liveness ▸ 82]      │
 │ [Style    ▸ 79]                                        │
 ├──────────────────────────────────────────────────────┤
 │ Poses                                                  │
 │ [neutral] [blink] [smile] [turn-l] [turn-r] [nod-up]   │
 │  썸네일 그리드 (클릭 → 프리뷰 플레이어에서 해당 포즈)  │
 ├──────────────────────────────────────────────────────┤
 │ Issues (3)                                             │
 │ • hair_side_r 의 광원 방향이 전체와 15° 차이           │
 │ • cloth_main 의 채도가 팔레트보다 8% 높음              │
 │ • smile_big 에서 mouth_inner 미노출                    │
 │ [Fix suggestions ▾]                                   │
 └──────────────────────────────────────────────────────┘
```

### 8.2 문구 규칙

- **항상 무엇이 왜 깎였는지**를 사용자 말로 번역. ("ΔE_mean=6.3" 은 내부 로그에만.)
- 각 이슈에는 **권장 액션** 이 붙는다: `재생성`, `수동 조정`, `그대로 사용`.
- 기술 세부는 "Advanced" 토글 뒤로 숨긴다.

## 9. 인간 리뷰 큐 (Human Review Queue)

### 9.1 두 종류

1. **사용자 리뷰**: 자기 아바타에 대해 본인이 수정. 에디터 UI.
2. **스튜디오 QA 리뷰**: 엔터프라이즈 계약에서, 조직 QA 리드가 팀원들의 아바타를 승인.

### 9.2 리뷰 화면 요구사항

- 프리뷰 플레이어 + 지표 패널 + 원본/비교.
- 파츠별 "이 부분 재생성" 원터치.
- 메모/승인/반려.

### 9.3 SLA

- 사용자 리뷰: 셀프, SLA 없음.
- 스튜디오 QA: 티어에 따라 영업일 24/12/4시간.

## 10. 회귀 테스트 & 골든셋

### 10.1 골든셋 구성

- 100 아바타 (다양한 스타일/체형/복식).
- 각 아바타는 모든 포즈의 기준 렌더 + 기대 점수 구간을 가진다.
- 매 PR / 파이프라인 배포 전에 자동 실행.

### 10.2 허용 변동

- 골든셋 전체 평균 점수 변화 ≤ 1.0 허용.
- 개별 아바타 점수 변화 ≤ 3.0 허용.
- 상한/하한 초과 시 **배포 블록**, 조사 필요.

### 10.3 시각 회귀

- 포즈별 참조 PNG 와 현재 PNG 를 **SSIM / LPIPS** 로 비교.
- LPIPS 차이 > 0.08 은 경고, > 0.15 은 블록.

## 12. 보안 / 개인정보

- 검수 결과 썸네일은 **사용자 소유** (우리는 권한 없음).
- 골든셋 아바타는 **합성/내부 소유** 만. 외부 사용자 아바타로 회귀 검증 금지.
- 익명 QA 용도로 사용자 아바타를 쓰려면 명시적 동의.

---

## 13. 관찰성

- 전체 아바타의 점수 히스토그램, 카테고리 분해.
- 실패 태그 상위 10 (예: `hair_clip_thru_face`, `iris_out_of_sclera`) 트렌드.
- 사용자 오버라이드 비율 (너무 높으면 게이트 기준을 의심해야 함).

---

## 14. 확장점

- **사용자 커스텀 지표**: PRO 이상에서 자체 지표 추가 (예: "캐릭터가 우리 브랜드 팔레트에서 벗어나면 감점").
- **플러그인 렌더러**: 특수 목적(AR 프리뷰, 3D 합성) 렌더 후 검수.

---

## 15. 열린 질문

- 영상 기반 검수(4초 렌더)는 비용이 크다. 기본은 정지 포즈 N장으로 두고, 영상은 옵션으로 제공?
- FaceID 같은 정체성 유사도 측정을 어떤 모델로 할지 (오픈소스 vs 상용). 저작권·편향 이슈.
- 스타일 임베딩 거리 임계값을 템플릿/스타일 카테고리별로 자동 학습할지.
