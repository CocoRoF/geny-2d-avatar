# 06. 후처리 파이프라인 (Post-Processing Pipeline)

> **한 줄 요약**: **후처리가 이 제품의 진짜 해자다.** AI 생성 파츠는 톤/선/광원/그림자가 파츠마다 다르다. 후처리는 이 불일치를 줄여 "한 사람이 그린 것처럼" 만든다.

---

## 1. 왜 후처리가 제품의 본질인가

### 1.1 관찰

- AI 를 슬롯별로 돌리면 거의 무조건 다음이 발생한다:
  1. 파츠마다 **채도·명도·색온도** 가 미묘하게 다르다.
  2. **선 굵기** 와 선의 "압" 이 다르다.
  3. **광원 방향** 이 제각각이다 (왼쪽 위 vs 오른쪽 위 vs 정면).
  4. **그림자 강도** 가 다르다.
  5. **알파 경계** 처리가 다르다 (하드 vs 안티앨리어스 vs 페더).
  6. **겹침 영역** 에서 중첩된 선이 보인다.
  7. 얼굴과 목의 톤이 **이어지지 않는다**.

### 1.2 결과

- 생성 직후 이미지는 **움직일 때 더 심하게 어색** 해진다. 정적 스크린샷에서는 "괜찮은 것 같다" 라고 속을 수 있다.
- 이 단계를 생략한 다른 시도들이 대부분 실패한 이유다.

### 1.3 해법의 형태

- **결정론적(Deterministic) 파이프라인** 으로 구성.
- 모든 단계는 **수치적으로 검증 가능한 목표치** 를 가진다 (주관이 아니라).
- 단계 하나를 꺼도 다음 단계가 정상 동작한다 (느슨한 결합).

---

## 2. 후처리의 6 단계 (Six-Stage Pipeline)

```
 [0] Preflight         ← 입력 파츠 세트 수집, 유효성 확인
  │
 [1] Alpha Sanitation  ← 알파 경계, 먼지, 구멍 정리
  │
 [2] Edge Unification  ← 선 두께/압/안티앨리어스 통일
  │
 [3] Color Normalize   ← 팔레트 락, 컬러 ΔE 정렬, 히스토그램 매칭
  │
 [4] Light & Shadow    ← 광원 방향 통일, 음영 강도 표준화
  │
 [5] Seam & Overlap    ← 파츠 간 이음매 보정, 중복 선 제거
  │
 [6] Pivot & Resample  ← 앵커 기준 좌표 정규화, 해상도 통일
```

각 단계는 **독립 노드** 로 DAG 에 등록된다. 순서는 위와 같이 고정.

---

## 3. Stage 0 — Preflight

### 3.1 하는 일

- 이 아바타의 필수 슬롯이 모두 채워졌는지.
- 각 파츠가 `spec.validation` 의 기하/알파 조건을 통과했는지.
- 부족하거나 위반 → **후처리 중단**, 생성 재시도 요청.

### 3.2 입력 수집

- 파츠 PNG + `parts/{slot}.meta.json` + `spec` + `color_context` 매핑.
- 스타일 프로파일 (팔레트, 톤 힌트).
- 원본 레퍼런스 이미지 (있으면) 의 **전역 통계** (mean luminance, dominant colors, edge histogram).

### 3.3 출력

- 정합된 파츠 묶음 + 각 파츠의 **전역 통계(statistics)** + 목표치 (targets).

---

## 4. Stage 1 — Alpha Sanitation

### 4.1 목표

- 깔끔한 경계. AI 가 남긴 "유령 픽셀" 제거.
- 외곽 페더를 스펙대로.
- 작은 구멍(고스트 알파) 제거.

### 4.2 단계별 연산

1. **Pre-multiplied → Straight** 변환 (필요 시).
2. **알파 임계 클리너**: `0 < α < 8` 인 "노이즈 알파" 는 0 으로.
3. **형태학적 닫힘(morphological close)** 로 1–2 픽셀 구멍 제거. 단, 의도된 구멍은 보존해야 하므로 **슬롯별로 max_hole_px** 한도.
4. **경계 에지 페더**: 스펙의 `alpha_edge_policy` 대로 (e.g., `feather_2px`).
5. **바깥 알파가 UV 박스를 벗어남** → 박스 밖을 0.
6. **바운딩박스 재계산** 후 메타에 기록.

### 4.3 자동 검증

- 외곽 1px 링에서 `min_alpha > 0 and max_alpha < 255` 인 픽셀 비율 ≥ 0.8 (페더가 고르게 되었는가).
- 구멍 수 ≤ 슬롯별 한도.

---

## 5. Stage 2 — Edge / Line Unification

### 5.1 목표

- **선 굵기** 가 파츠 간 일치.
- **선의 질감** (거친지/매끈한지) 이 일관.
- **선 색** 이 통일 (순흑 vs 진갈색).

### 5.2 접근

1. **선화 추출**: XDoG / 학습 기반 line extractor 로 선 맵을 얻는다.
2. **선폭 측정**: 거리 변환(distance transform)으로 선폭 분포 추출.
3. **목표 선폭**: 템플릿 스펙의 `line_weight_hint_px` (예: 2.5 px @2048).
4. **정규화**:
   - 너무 굵음 → 선 폭 축소(morphological erosion + smoothing).
   - 너무 얇음 → 가중 두께 증가(guided dilation).
   - 끊어진 부분 → 선 연결 모델 or 휴리스틱.
5. **선 색 통일**: 전체 아바타 기준 평균 선 색에 맞춤 (순흑이면 #1a1a22 같은 정해진 값).
6. **재합성**: 원본 색 레이어 × 보정된 선 레이어.

### 5.3 주의

- **얼굴 선은 보수적으로**. 얼굴의 정체성을 파괴할 수 있으므로 변경량 상한을 slot 별로 다르게.
- `line_weight_policy: "preserve"` 플래그가 켜져 있으면 이 단계 skip.

---

## 6. Stage 3 — Color Normalize

### 6.1 목표

- 같은 **color_context** 안의 파츠는 통계적으로 "비슷해 보이게".
- 스타일 프로파일의 팔레트 락이 있으면 그에 맞게.
- 극단값(클리핑)과 채도 폭주를 제어.

### 6.2 수학적 정의

각 context `C` 에 대해:

- 관측 통계: 평균 Lab `μ_obs`, 공분산 `Σ_obs`.
- 목표 통계: `μ_target`, `Σ_target`.
- 변환: Cholesky-based color transfer (Reinhard 확장) — 각 파츠를 `T(x)` 로 이동.

### 6.3 목표의 결정

1. 스타일 프로파일에 **명시 팔레트** 가 있으면 → 팔레트 중심을 `μ_target`.
2. 없으면 → 해당 아바타의 **지배 파츠**(가장 넓은 면적, 예: face_base/cloth_main) 의 통계를 목표로.
3. 지배 파츠가 불안정하면 → 원본 레퍼런스 이미지의 해당 context 영역 통계.

### 6.4 팔레트 락 (Palette Lock)

- 팔레트 잠금 모드에서는 생성물의 지배색을 **k-means 4** 로 추출한 뒤, 최근접 팔레트 색으로 **이동**.
- 이동량 상한: ΔE < 12. 초과 시 경고.

### 6.5 히스토그램 매칭 (Optional)

- 선택적으로 **Y 채널(luminance) 히스토그램** 만 매칭 — 색조는 건드리지 않음.
- `cloth_main` 같이 질감 보존이 중요한 파츠에만.

### 6.6 자동 검증

- 같은 context 내 `ΔE_mean` ≤ 5.
- 지배색의 팔레트 이탈 ≤ 12.
- 채도 상한 S ≤ 사용자 설정값.

---

## 7. Stage 4 — Light & Shadow

### 7.1 목표

- **광원 방향** 이 모든 파츠에서 같은 방향.
- **그림자 강도** 가 context 별로 일관.
- **하이라이트 위치** 가 얼굴을 포함해 일관.

### 7.2 광원 추정(Lighting Estimation)

- 간단 구근 모델(spherical gradient) 피팅:
  - 각 파츠에서 밝기 그라디언트 벡터 추정.
  - 주요 파츠 (face_base, cloth_main) 의 그라디언트를 **가중 평균** → 전역 광원 방향 `L*`.
- 단독 파츠의 광원 방향이 `L*` 와 크게 어긋나면 → **음영 레이어 재합성**.

### 7.3 음영 재합성

- 원본 이미지에서 **알베도(albedo) + 음영(shading)** 을 분리 (지능형 intrinsic decomposition).
- 음영을 제거하고, 전역 광원 `L*` 기반의 **표준 음영** 을 합성.
- 품질이 미묘해질 수 있어 **강도 파라미터** 를 둔다: `relight_intensity ∈ [0,1]`.

### 7.4 얼굴 음영의 특수 처리

- 얼굴은 고개를 회전하면 음영이 따라와야 한다.
- 생성 단계에서 `face_shadow` 를 **평평한(flat) 텍스처** 로 유지하고, 런타임에 디포머로 음영 강도를 조절.

### 7.5 그림자 강도 표준화

- 각 context 의 "그림자 영역(shadow region)" 을 포함하는 어두운 픽셀의 평균 명도 차이(Δ `L*`)를 측정.
- 목표 Δ `L*` = 15–20 (중간 대비).
- 초과/미달 시 음영 레이어 게인 조정.

### 7.6 자동 검증

- 광원 방향 오차 ≤ 15°.
- context 별 Δ `L*` 편차 ≤ 5.

---

## 8. Stage 5 — Seam & Overlap

### 8.1 목표

- 파츠 경계에서 **선 겹침, 틈, 톤 차이** 제거.
- 머리카락-얼굴, 옷깃-목, 눈-얼굴 경계 특히 중요.

### 8.2 대표 케이스

1. **머리카락 ↔ 얼굴**
   - `hair_front` 의 밑면과 `face_base` 의 윗면이 겹치는 영역.
   - 머리카락은 **알파 그라디언트** 로 얼굴과 블렌딩. 1–3 px 이행 구간.
2. **옷 ↔ 목 / 팔**
   - `cloth_main` 하단 넥 라인 ↔ `neck` 상단.
   - 겹침부에서 선이 두 번 그려진 것처럼 보이면 → 선화 레이어에서 그쪽만 **에지 서브트랙션**.
3. **눈 ↔ 얼굴**
   - 눈꺼풀 라인이 `face_base` 의 눈 영역과 맞물림.
   - 얼굴 쪽에 미리 **눈 자국(alpha hole)** 을 파두고, 눈 파츠가 그 위에.

### 8.3 방법

- **경계 추출**: 각 파츠의 외곽과, 인접 파츠(의존 관계) 외곽을 2 px 확장하여 교집합을 얻는다.
- **교집합 영역 분석**:
  - 선의 중첩도 → 에지 서브트랙션.
  - 색 점프 → 가우시안 혼합 + 로컬 히스토그램 매칭.
  - 알파 점프 → 알파 그라디언트 재설정.
- **우선순위**: 근거(foreground) 파츠가 위로 — 머리카락 > 얼굴. 스펙의 `z_order` 를 존중.

### 8.4 자동 검증

- 경계 2 px 링에서 색 ΔE 최대값 ≤ 12.
- 중첩 선 카운트 0 (머리카락-얼굴 경계에서 2 중 선 픽셀이 없어야 함).

---

## 9. Stage 6 — Pivot & Resample

### 9.1 목표

- 모든 파츠를 **표준 캔버스 좌표** 로 정렬.
- 해상도·비율·DPI 를 통일 (2048 HD / 4096 PRO).
- 각 파츠의 **앵커 좌표가 스펙과 일치**.

### 9.2 앵커 보정

1. 파츠의 현재 앵커 탐지 (검출기).
2. 스펙 기대 앵커와의 오프셋 계산.
3. 평행 이동 + (필요 시) 균일 스케일로 정렬.
4. 이동/스케일이 허용치를 초과하면 → 재생성 요청으로 돌아감.

### 9.3 해상도 계단

- 기본: 2048.
- PRO: 4096. 단, 업샘플 시 해상도 노이즈가 발생하므로 **Real-ESRGAN 계열** 업스케일러로 세분화.

### 9.4 서브픽셀 정렬

- 회전 없이 평행 이동만 수행. 서브픽셀 보간은 **Lanczos3** 기본.
- 회전이 필요하면 **생성 단계에서 각도를 재지정**, 후처리에선 회전 금지 (중복 품질 손실).

### 9.5 결과 포맷

- PNG 8 bit/channel + alpha. sRGB.
- 사이드카: `parts/{slot}.meta.json` 의 `geometry` 갱신.

---

## 10. 선택적(Optional) 후처리 모듈

핵심 6 단계 외에 다음을 상황별로 활성화.

### 10.1 Denoise

- 생성 노이즈가 크면 적용. AI 가 선명한 경우 skip.
- 과도하면 디테일 손실 — 기본 OFF.

### 10.2 Super-Resolution

- 2048 → 4096 업스케일.
- 메모리·시간 비용이 큼. 사용자 선택(PRO) 또는 export 단계에서만.

### 10.3 Style Embedding Alignment

- 임베딩 벡터 공간에서 스타일 이탈한 파츠를 **스타일 projection** 으로 끌어당김.
- 아직 실험적. α 에서 shadow 적용.

### 10.4 Line Color Unification

- 모든 파츠의 선 레이어 색을 지정된 hex 로 강제.
- 애니메 풍 표준(#1a1a22) 프리셋 제공.

---

## 11. 각 단계의 실패 처리

| 단계 | 실패 조건 | 처리 |
|---|---|---|
| Alpha | 경계 복구 불가 | 해당 파츠 재생성 |
| Edge | 선 추출 실패 | 이 단계 스킵(보수), 경고 기록 |
| Color | ΔE 과도 | 재시도(가중치 변경), 실패 시 사용자 확인 |
| Light | 광원 추정 신뢰도 낮음 | 음영 재합성 스킵, 원본 유지 |
| Seam | 재현성 없음 | 사용자에게 "Human Review" 제안 |
| Pivot | 앵커 신뢰도 낮음 | 재생성 + 사용자 수동 앵커 지정 UI 노출 |

---

## 12. 파이프라인 결정성 (Determinism)

- 같은 입력 + 같은 버전 → 같은 출력. 모든 난수는 고정 시드.
- 버전은 `post_proc@{major.minor.patch}` 로 메타에 저장.
- 후처리 재실행은 어떤 시점이든 가능해야 한다 (원본 파츠 보존).

---

## 13. 성능 목표 (Performance SLOs)

파츠 1개 기준 (2048 캔버스):

| 단계 | 목표 시간 | 하드웨어 |
|---|---|---|
| Alpha | ≤ 80 ms | CPU |
| Edge | ≤ 300 ms | CPU (옵션 GPU) |
| Color | ≤ 150 ms | CPU |
| Light | ≤ 500 ms | CPU (옵션 GPU) |
| Seam | ≤ 250 ms | CPU |
| Pivot | ≤ 100 ms | CPU |
| 합계 | ≤ 1.5 s | |

27 파츠 아바타 1대 후처리 총합 목표: **≤ 15s (병렬)**, **≤ 45s (직렬 상한)**.

---

## 14. 관찰성(Observability)

- 각 단계 전/후 **썸네일 저장** (디버그 모드).
- 통계 메트릭 기록: 각 단계가 만든 ΔE, Δ L*, 선폭 분포.
- 실패/경고는 아바타의 "health panel" 에서 사용자에게 보임 (숨기지 않는다).

---

## 15. 테스트 / 골든셋 (Post-Processing Golden Set)

- 모든 알고리즘 변경은 **200 파츠 골든셋** 에서 회귀 검증.
- 주요 지표: ΔE_mean, Δ L*, 선폭 표준편차, 경계 노이즈 비율.
- 배포 게이트: 골든셋 전부에서 회귀 ≤ 5%.

---

## 16. 열린 질문

- Relighting 알고리즘을 내부 구현할지, 3rd-party 모델에 맡길지 (얼굴 붕괴 리스크).
- ~Color transfer 에서 Lab vs OKLab 선택~ → 세션 32 에서 **CIE 1976 L*a*b* (D65)** 로 결정. `@geny/post-processing` 의 `colorSpace: "lab"` 옵션이 Reinhard 채널별 재매핑을 Lab 공간에서 수행. OKLab 은 필요 시 별도 옵션으로 후속 추가.
- 후처리 파이프라인 자체를 **사용자 커스터마이즈** 가능하게 노출할지 (PRO 기능 후보).
