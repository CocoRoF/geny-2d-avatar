# 00. 프로젝트 목표

## 한 줄 요약

**프리셋 뼈대 + 새 텍스처 = 완성된 Live2D 모델**.
사용자가 웹 UI 에서 프리셋 하나를 고르고 텍스처를 생성·업로드하면, 합쳐진 Live2D 번들을 즉시 프리뷰·다운로드할 수 있는 파이프라인.

## 핵심 등식

```
   texture (생성 또는 업로드)
 + preset  (기존 프리셋 뼈대 — mao_pro 포함, 여러 개 중 선택)
 ─────────────────────────────
 = 완성된 Live2D 모델 (.moc3 기반 번들)
```

## 핵심 원칙

1. **뼈대는 만들지 않는다 — 프리셋을 고른다.**
   모든 뼈대 (mao_pro 의 것이든, 우리가 Cubism Editor 로 저작한 것이든) 는 **동등한 프리셋**이다. 런타임에 뼈대를 조립하지 않는다.

2. **mao_pro 는 참고인 동시에 프리셋이다.**
   Live2D 공식 샘플 `mao_pro_ko` 의 뼈대는 (a) 다른 프리셋 저작의 설계 참고일 뿐 아니라 (b) **그 자체로 등록된 첫 프리셋**이다. `rig-templates/base/mao_pro/` 항목으로 편입.

3. **프리셋은 많을수록 좋다.**
   halfbody / fullbody / chibi / mao_pro / 사용자 저작 … 캐릭터 타입별·테마별 프리셋이 얼마든지 추가될 수 있는 카탈로그 구조.

4. **가변 자산은 텍스처뿐이다.**
   프리셋은 고정, 텍스처만 생성·교체. 이 원칙이 파이프라인 전체 결정론을 가능하게 한다.

5. **제품은 Web UI 다.**
   사용자 진입점은 브라우저. 프리셋 선택 → 텍스처 생성/업로드 → 프리뷰 → 다운로드가 전부 웹에서 끝난다. CLI 는 내부 도구일 뿐 제품이 아니다.

## 입·출력

### 입력 (웹 UI 기준)

- **프리셋 선택**: 카탈로그에서 하나 (예: `mao_pro@1.0.0`, `halfbody@1.3.0`)
- **텍스처 소스** (택 1):
  - (A) **AI 생성**: 프롬프트 + 시드 + (옵션) 레퍼런스 이미지·팔레트 힌트
  - (B) **수동 업로드**: 사용자가 제작한 PNG 파일 (프리셋의 atlas UV 규격 준수)
  - (C) **기존 텍스처 조정**: 프리셋의 base.png 를 로드해 색상·명도만 변경

### 출력

1. **`texture.png`** — 프리셋의 atlas UV 에 정합하는 RGBA PNG
2. **`avatar.json`** — 프리셋 + 텍스처 결합 기술서 (`schema/v1/avatar-metadata.schema.json`)
3. **Live2D 런타임 번들** (zip) — Cubism SDK 가 그대로 로드 가능한 파일 집합:
   - `model3.json`, `physics3.json`, `pose3.json`, `cdi3.json`, `motion3.json` × N, `exp3.json` × N
   - 참조하는 `.moc3` (프리셋에서 가져옴) + `texture_00.png` (새로 생성된 것)
4. **Web Avatar 번들** (JSON) — `<geny-avatar>` 가 직접 로드하는 프리뷰용 포맷

## 스코프

### IN ✅

- **프리셋 카탈로그** — mao_pro 포함 N 개. 각각 `rig-templates/base/<id>/<version>/` 디렉토리로 제공.
- **텍스처 생성** — AI 단일/슬롯별 + 수동 업로드 + 기존 텍스처 리컬러.
- **번들 조립** — 프리셋 뼈대 자산 + 새 텍스처 → Cubism / Web Avatar 번들 (결정론적).
- **웹 UI** — 프리셋 선택, 텍스처 입력, 실시간 프리뷰, 다운로드. 반응형.
- **백엔드 서비스 (최소)** — AI 호출 중계, 번들 생성 작업, 다운로드 서빙.
- **프리셋 추가 절차** — Cubism Editor 로 저작 → 파일 트리 분해 → 카탈로그 등록.

### OUT ❌

- 새 뼈대(드로어블·디포머·파라미터) 를 코드로 생성·조립
- 모션·표정 신규 저작 (프리셋에 묶여 제공됨)
- "파츠를 AI 가 새로 그리고 리그에 자동 피팅" (이전 스코프의 잔재 — 이 프로젝트는 **파츠가 아닌 텍스처**를 생성)
- 3D / VRM / MetaHuman 등 Live2D 외 플랫폼
- 라이선스 마켓플레이스·결제·SaaS 멀티테넌트 (별도 프로젝트 영역)

## 성공 기준 (측정 가능)

| 기준 | 의미 | 측정 방법 |
|---|---|---|
| **S1. Manual E2E** | 수동 업로드 텍스처 1장 + mao_pro 프리셋 → 웹 프리뷰에서 파라미터/모션/표정 전부 정상 | 수동 smoke test (브라우저) |
| **S2. Multi-Preset Swap** | 동일 텍스처(또는 대응 텍스처) 로 2 개 이상 프리셋 선택 전환 시 렌더 깨짐 0 | 웹 UI 에서 preset 드롭다운 변경 |
| **S3. AI Single-Gen** | 프롬프트 1 개 → AI 가 atlas 1장 생성 → 프리뷰 | 웹 UI 에서 "Generate" 버튼 end-to-end |
| **S4. Partial Regen** | 특정 슬롯(예: 머리) 만 재생성 → 나머지 유지 | UI 에서 파츠 선택 + regenerate |
| **S5. Deterministic** | `(preset_id, prompt, seed, adapter)` 동일 → 바이트 동일 텍스처 | CI 회귀 |
| **S6. Download** | 웹 UI 에서 번들 zip 다운로드 → 외부 Cubism Viewer 에서 정상 재생 | 수동 검증 |
| **S7. Web UI 접근성** | 비개발자가 README 없이 5 분 내 S1 완수 | 사용성 테스트 |

## 용어

| 용어 | 정의 |
|---|---|
| **뼈대 / 리그 (rig / skeleton)** | Live2D 의 드로어블·디포머·파라미터·물리 구조. `.moc3` 으로 컴파일됨. **이 프로젝트는 뼈대를 저작하지 않고 프리셋으로만 소비**. |
| **프리셋 (preset)** | 완성된 뼈대 + 모션 + 표정 번들. mao_pro 의 뼈대도 프리셋 중 하나. `rig-templates/base/<id>/<version>/` 디렉토리. |
| **3rd-party preset** | Live2D Inc. 등 외부에서 제공된 완제품 (예: mao_pro). |
| **derived preset** | Cubism Editor 로 우리가 직접 저작한 프리셋 (예: halfbody v1.3.0). |
| **아틀라스 (atlas)** | 프리셋이 규정한 UV 맵 (여러 파츠 → 한 장 PNG). |
| **슬롯 (slot)** | 아틀라스 안의 사각 영역. `part_id` 와 1:1 매핑. |
| **텍스처 (texture)** | 아틀라스를 채우는 PNG 이미지. **이 프로젝트가 생성·교체하는 유일한 가변 자산**. |
| **아바타 (avatar)** | 프리셋 + 텍스처 조합. `avatar.json` 으로 기술. |
| **번들 (bundle)** | Cubism 또는 Web Avatar 런타임이 직접 로드하는 파일 집합. |
