## 3. Design decisions

### D1 — `cp -R v1.2.0 v1.3.0` 후 덧칠
ADR 0003 이 **SemVer 디렉터리 병렬 유지** 를 강제하므로 v1.2.0 을 뜯어 고칠 수 없다. 가장 단순한 재현 가능 저작은 "v1.2.0 을 통째로 복사한 뒤 diff 로만 수정". 이 덕에 "v1.3.0 이 v1.2.0 과 어디서 갈라졌는지" 가 `git diff v1.2.0 v1.3.0` 으로 명확하며, 회귀 영향은 v1.3.0 디렉터리로 국한된다. 기존 4 버전 (v1.0.0 / v1.1.0 / v1.2.0 / v1.3.0) 번들 golden 은 전부 그대로.

### D2 — `body_breath_phys` 파라미터 이름 (왜 `_phys` 접미사인가)
초안에서는 `body_breath_offset` 을 고려했다. 하지만 `parameters.schema.json` 의 `physics_output` 파라미터 이름 regex 는 `_(sway|phys|fuwa)(_[lr])?$` 로 고정돼 있어(v1.2.0 시점에 세션 07 이 박았다) `_offset` 접미사를 거부한다. 세션 27 migrator TODO 본문이 `body_breath_phys` 로 표기한 것도 이 규약과 일치 — 따라서 규약을 그대로 따르고 파라미터 이름을 `body_breath_phys` 로 최종 확정. 이 선택은 **출력 파라미터 명명의 의미론적 단위가 `_phys` (물리 2차 출력), `_sway` (흔들림), `_fuwa` (볼륨)** 라는 docs/03 §6.2 규약과도 깨끗이 들어맞는다. 나중에 breath 계열이 늘어도 `_phys` 아래로 묶인다.

### D3 — `accessory_sway_phys` 는 **1 Setting / 2 출력 공유**
mao_pro PhysicsSetting9–12 (모자/리본/깃털/모자 위) 는 4 개 setting 으로 쪼개져 있지만 halfbody 의 `accessory_back` / `accessory_front` 는 같은 body 움직임에 반응하는 장신구 레이어라 **하나의 시뮬레이션으로 두 출력을 동시에 내는** 편이 자연스럽다. Cubism 물리 포맷은 한 PhysicsSetting 의 `output[]` 에 여러 destination_param 을 둘 수 있으므로, `accessory_sway_phys` 하나에 `accessory_back_sway` (scale 1.0 / weight 80) + `accessory_front_sway` (scale 0.8 / weight 70) 두 출력을 달았다. 계산 비용은 setting 하나, 튜닝은 출력별 scale/weight 차등. 결과적으로 머리 8 + 옷 4 = 12 라는 `normal` 프리셋 목표를 과하게 넘지 않으면서 2 파츠를 독립 제어.

### D4 — `ahoge_warp` 는 `head_pose_rot` 자식
ahoge(안테나 머리) 는 머리 회전을 따라가야 하되 자체 sway 물리로 지연/오버슈트가 커야 캐릭터성을 낸다(docs/03 §12.1 #2 vibe "상반신 정면 + 친근한 흔들림"). 따라서:
- deformer 부모: `head_pose_rot` (머리 회전 상속)
- physics 입력: `head_angle_x` · `head_angle_y` · `body_angle_x` (빠른 회전에 antenna 가 뒤따라 흔들리도록)
- physics 출력 scale 1.2 (원값보다 과장) + radius 5 + mobility 1.0 (가장 자유로움) + delay 0.55
이렇게 두면 "머리는 멈춰도 ahoge 는 한 박자 늦게 멈춘다" 는 시각 효과가 자연스럽다. `head_pose_rot` 자식이라는 위치는 migrator TODO 에도 적혀 있던 **유일한 저자 판단** 이었고, 앞머리 sway warp 와 같은 층위에 두었다 (이 경우 `hair_front_warp` sibling).

### D5 — `head_angle_y` / `body_angle_y` 를 뒤늦게 `physics_input: true` 로
v1.0.0~v1.2.0 까지는 이 두 파라미터가 물리 입력으로 쓰이지 않아 플래그가 없었다. v1.3.0 에서 PhysicsSetting10 (`ahoge_sway_phys`) 이 `head_angle_y` 를, PhysicsSetting12 (`body_breath_phys`) 가 `body_angle_y` 를 입력으로 참조하면서 validator 가 "physics input 으로 선언 안 된 파라미터를 참조" 로 거부했다. **v1.3.0 의 parameters.json 에만** 플래그를 추가 (v1.2.0 / v1.1.0 / v1.0.0 파일은 건드리지 않음 — 디렉터리 격리 덕에 가능). 이 fixup 은 문서화된 의미 변경이 아니라 "늘 물리 입력이었어도 됐는데 아직 아무도 쓰지 않았을 뿐" 이므로, 하위 버전들은 자신의 물리 팩이 실제로 참조하지 않는 이상 플래그를 안 달고 살아도 괜찮다.
