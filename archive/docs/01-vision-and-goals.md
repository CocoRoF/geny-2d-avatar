# 01. 비전과 목표 (Vision & Goals)

> **"2D 아바타 제작을, 한 명의 전문가가 2주 걸리던 일에서 한 명의 크리에이터가 2시간 안에 끝내는 일로 바꾼다."**

---

## 1. 비전 (Vision Statement)

**geny-2d-avatar는 "2D 캐릭터 리깅의 Figma" 가 되는 것을 목표로 한다.**

- Figma 이전의 UI 디자인: 포토샵 + 개별 파일 + 디자이너 1인 의존 → 협업/재사용/스케일링이 어려움
- Figma 이후: 컴포넌트 라이브러리, 협업, 버전 관리, 디자인 시스템 → 프로덕트 디자인이 민주화됨

Live2D / Spine 기반 2D 아바타 시장은 **Figma 이전 상태**이다:

- Cubism Editor 숙련자 1인이 2–4주 수작업
- 파츠 구성·명명 규칙이 제작자마다 다름
- 재사용/파생이 사실상 불가능
- 새 캐릭터를 만들려면 처음부터 다시 그려야 함

geny-2d-avatar는 이 시장을 **"템플릿 + 파츠 + AI 재설계 + 검수" 의 반자동 워크플로우** 로 재구성한다.

---

## 2. 문제 정의 (Problem Statement)

### 2.1 현재 시장의 3대 고통

| # | 고통 | 현상 | 영향 |
|---|---|---|---|
| P1 | **제작 병목** | 리거(rigger) 1인당 캐릭터 1대당 80–200시간 | VTuber 지망자 대기 줄, 인디 게임 개발 지연, 외주 단가 $500–$5,000 |
| P2 | **비재사용성** | 캐릭터마다 파츠 구조·네이밍·파라미터가 달라 라이브러리화 불가 | 의상 변경 하나도 재제작, 시즌 이벤트 대응 불가 |
| P3 | **일관성 붕괴** | AI로 무작위 생성 시 톤/선/광원/그림자가 파츠마다 달라짐 | "프롬프트로 VTuber 모델" 같은 시도가 모두 장난감에 그침 |

### 2.2 이 문제가 풀리지 않는 이유

- **"생성 AI" 접근의 실패**: 모델에게 "Live2D 모델을 만들어라" 고 하면 파츠 분리가 되지 않는 단일 이미지, 혹은 애니메이션이 불가능한 형태를 내놓는다.
- **표준 부재**: Cubism SDK는 파일 포맷은 표준화되어 있지만 **"좋은 리그 구조" 의 규격**이 없다.
- **검수의 부재**: 파츠 단위 생성물은 정적으로는 괜찮아 보여도, blink/mouth/angle 같은 변형에 들어가면 깨진다.

### 2.3 이 프로젝트가 해결하는 방식

- 리그·파츠 구조를 **강제된 표준**으로 고정.
- AI는 **"자유 생성" 이 아니라 "제약된 파츠 재설계"** 만 수행.
- 후처리로 일관성을 복원.
- 테스트 포즈 세트로 **움직일 때의 품질**을 자동 검수.
- 인간은 "전부 만드는 사람" 에서 "마지막 10% 를 다듬는 사람" 이 된다.

## 3. 타겟 사용자 (Personas)

> 페르소나는 "허구의 평균 사용자" 가 아니라 **의사결정 트레이드오프를 가를 때 기준이 되는 프로토타입** 이다. 모든 기능은 "이 페르소나들에게 어떤 가치를 주는가" 로 검증한다.

### 3.1 Primary Persona — "인디 VTuber 지망 크리에이터" (Nari, 22)

- **상황**: 유튜브/트위치에서 VTuber로 시작하고 싶음. 예산 $50–$300, 기한 2주.
- **고통**:
  - 외주 의뢰 시 최소 $800–$3,000 + 6–10주 대기
  - Live2D Cubism 직접 배우기엔 러닝 커브가 가파름
  - 기성 무료/저가 모델은 "나의 캐릭터" 가 아님
- **원하는 것**:
  - 자신의 캐릭터 일러스트 하나 + 취향 프롬프트로 바로 아바타 생성
  - VTube Studio 로 즉시 송출 가능한 .model3.json
  - 의상/머리 바꾸기가 쉬움
- **우리의 답**: 원-클릭 스타트 프리셋 + 빠른 프리뷰 + VTS 직접 내보내기

### 3.2 Primary Persona — "인디 게임 스튜디오 아트 리드" (Dohee, 31)

- **상황**: 3–5인 스튜디오, 2D 비주얼 노벨 또는 소셜 게임. NPC 20–60명 필요.
- **고통**:
  - 60명을 전부 리깅하면 $30k+ 예산 필요, 일정 슬립
  - 캐릭터마다 스타일이 미묘하게 달라져 브랜드 통일감 훼손
  - 엔진 (Unity/Unreal) 통합 시 포맷 컨버전 이슈 반복
- **원하는 것**:
  - 스튜디오 스타일 가이드를 고정한 채 60명 찍어내기
  - Unity 프리팹으로 바로 들어오는 포맷
  - 의상/표정 세트를 프로그래매틱하게 교체
- **우리의 답**: 스튜디오 계정 + 스타일 프로파일 잠금 + 배치 생성 + Unity 플러그인

### 3.3 Secondary Persona — "IP 라이선싱 운영자" (Kenji, 44)

- **상황**: 기존 만화/웹툰 IP를 버튜버·챗봇·광고 영상으로 확장.
- **고통**:
  - 원작 작가 스타일로 2D 모델을 만드는 데 수백만 원
  - 여러 플랫폼 대응 포맷 제작/관리 비용
  - 저작권/파생물 관리가 흩어져 있음
- **원하는 것**:
  - 원작 스타일 보존 + 다수 캐릭터 동시 제작
  - 버전/파생물 계보 추적
  - 권리 설정/정산
- **우리의 답**: IP 작업공간(workspace) + 스타일 락 + 파생물 계보 + 엔터프라이즈 계약

### 3.4 Tertiary Persona — "교육/커머스 챗봇 운영자" (Amy, 29)

- **상황**: 고객 응대/학습 보조 챗봇에 "말하는 캐릭터" 필요. 리깅 지식 0.
- **원하는 것**: 템플릿 하나 선택 → 회사 로고/컬러 반영 → Web SDK 한 줄로 붙이기.
- **우리의 답**: 프리-리그드(pre-rigged) 템플릿 카탈로그 + 스타일 토큰 + `<geny-avatar>` 웹 컴포넌트.

### 3.5 Non-Persona (의도적으로 타겟하지 않는 사람)

- **프로 Cubism 스튜디오**: 이미 자체 파이프라인 보유. 우리 툴을 쓰면 자유도가 떨어짐. 대신 "커뮤니티 템플릿 제공자" 로 참여 유도.
- **3D VTuber 사용자**: 범위 밖. 별도 제품으로 분리.
- **완전 무료·무제한 기대 사용자**: BM 충돌. 무료 티어는 제한적으로만 제공.

## 4. 핵심 가치 제안 (Value Proposition)

### 4.1 한 줄

> **"자기 캐릭터 일러스트와 프롬프트로, 30분 안에 움직이는 2D 아바타를 갖는다. 품질은 외주의 70–85%, 가격은 1/20, 시간은 1/50."**

### 4.2 가치의 3층 구조

```
  ┌──────────────────────────────────────────┐
  │ L3. 배포 & 운영 가치                      │
  │   "만든 것을 어디서든 쓸 수 있다"          │
  │    VTS, Web, Unity, Unreal, Mobile        │
  ├──────────────────────────────────────────┤
  │ L2. 커스터마이징 가치                      │
  │   "내 캐릭터의 정체성이 유지된다"          │
  │    스타일 락, 파츠 단위 수정, 버전 계보    │
  ├──────────────────────────────────────────┤
  │ L1. 생산성 가치                           │
  │   "혼자서도, 빠르게, 싸게"                │
  │    AI 재설계 + 표준 리그 + 자동 검수       │
  └──────────────────────────────────────────┘
```

### 4.3 경쟁 대안과의 비교

| 비교 대상 | 현재 대안의 한계 | 우리의 차별점 |
|---|---|---|
| **Cubism Editor 직접 제작** | 시간·학습비용 높음, 재사용 불가 | 자동화 + 템플릿화 |
| **외주 의뢰** | 비쌈, 느림 | 1/20 가격, 1/50 시간 |
| **무료 공용 모델 사용** | 내 캐릭터가 아님 | 자기 IP를 보존 |
| **"프롬프트 → Live2D" 를 주장하는 신생 AI 툴** | 품질 불안, 애니메이션 깨짐 | 파츠 재설계 + 후처리 + 검수로 움직일 때 안 깨짐 |
| **Ready Player Me 류 (3D)** | 2D 일러스트 감성이 아님 | 애니/일러스트 팬덤 문법에 최적화 |

### 4.4 차별점의 방어선 (Moats)

- **데이터 자산**: 베이스 리그 라이브러리, 파츠 정렬 키포인트 데이터셋, 스타일 프로파일 라이브러리.
- **후처리 파이프라인**: 색/선/광원 통일 로직은 복제 난이도가 높다.
- **검수 렌더러**: "움직일 때 깨지는지" 를 정량화하는 평가기는 시장에 거의 없다.
- **커뮤니티**: 사용자 기여 템플릿·의상·표정 팩 마켓 (16 BM 참조).

## 5. 목표 (Goals)

### 5.1 북극성 지표 (North Star Metric)

**NSM = "주간 성공 아바타 완성 수 (Weekly Successful Avatar Completions, WSAC)"**

- 정의: 주간 동안 사용자가 생성 → 내보내기(export) 단계까지 완료한 아바타 수. 환불/삭제는 제외.
- 선정 이유:
  - 사용자의 실제 "끝까지 간 성공 경험" 을 센다. (가입자 수, 생성 시도 수 같은 허영 지표가 아님)
  - 매출, 리텐션, 추천 전환과 상관이 가장 높을 것으로 예상 (α 기간 검증).
- 목표:
  - α 종료 시점: WSAC 200
  - β 종료 시점: WSAC 2,000
  - GA 후 6개월: WSAC 20,000

### 5.2 보조 지표 (Supporting Metrics)

| 카테고리 | 지표 | 측정 방법 | β 기준 목표 |
|---|---|---|---|
| **속도** | 신규 사용자가 첫 export 까지 걸린 시간 (TTFE) | p50 / p90 | p50 ≤ 30분, p90 ≤ 2시간 |
| **완주율** | 생성 시작 → export 성공 전환율 | 퍼널 | ≥ 45% |
| **품질** | 검수 렌더러 자동 점수 (0–100) | 8번 문서 기준 | 평균 ≥ 78 |
| **재방문** | WAU / MAU | 코호트 | ≥ 0.4 |
| **수익** | 유료 전환율 | 무료 → 유료 | ≥ 6% |
| **비용** | 아바타 1대당 AI 호출 비용 | 로그 | ≤ $0.60 |
| **신뢰** | 환불/불만 비율 | CS 티켓 | ≤ 2% |

### 5.3 질적 목표 (Qualitative)

- 신규 사용자의 **"처음 30초 경험" 이 마술적이어야 한다**: 템플릿 클릭 → 자기 캐릭터 1장 업로드 → 5초 안에 "될 것 같다" 확신.
- 파워 유저의 **"30번째 아바타" 경험이 여전히 좋아야 한다**: 반복 작업이 피곤하지 않도록 배치/프리셋.
- **실패해도 안전한 경험**: 생성이 망했을 때 "왜 망했는지" + "어떻게 고치는지" 를 말해주는 피드백.

## 6. 비목표 (Non-Goals) — 의도적으로 하지 않는 것

**중요**: 이 항목들은 "언젠가" 가 아니라 **"적어도 GA까지는 하지 않는다"** 로 선언한다.

1. **3D 아바타 지원** — 별도 제품.
2. **자체 이미지 생성 모델의 프리트레인** — 우리는 오케스트레이터·파이프라인 회사다.
3. **범용 일러스트 편집기** — Photoshop/Clip Studio 를 대체하지 않는다.
4. **실시간 모션 캡처 SDK 개발** — VTube Studio 등 기존 SDK에 연동만.
5. **완전 무료 무제한** — 무료 티어는 학습용/체험용으로만.
6. **완전 자동화 주장** — 마케팅 카피에도 "반자동" 을 명시. 기대치 관리.
7. **완전 맞춤 리깅(Full Custom Rigging)** — 템플릿에서 벗어난 자유 리깅은 전문 툴에 맡긴다. (플러그인 API는 열어둔다.)

## 7. 제품 원칙 (Product Principles)

설계·기획 회의에서 찬반이 갈릴 때 **투표가 아니라 원칙으로 결정한다**.

1. **Standards beat cleverness.** 영리한 한 번보다 표준 100번이 강하다.
2. **Make the happy path absurdly fast.** 즐거운 경로 p50 1분, 에지 케이스는 p99 1시간 허용.
3. **If it can break in motion, it must be tested in motion.** 움직이면 깨질 수 있는 것은 움직이며 테스트한다.
4. **AI is a contractor, not a creative director.** AI는 지시받는 계약자일 뿐, 디렉터는 사용자와 시스템이다.
5. **Humans finish, machines start.** 기계가 시작하고 인간이 끝낸다.
6. **Every artifact must be rolled back.** 모든 결과물은 버전·롤백·복원 가능해야 한다.
7. **Visible uncertainty beats invisible failure.** 신뢰도 낮으면 숨기지 말고 표시한다.
8. **Quality gates are non-negotiable.** 검수 점수 미달은 배포되지 않는다. 단 override는 로그에 남긴다.

## 9. 위험 요약 (Risks at a Glance)

상세는 [17-risks-and-mitigation.md](./17-risks-and-mitigation.md) 참조.

- **법적**: 생성된 스타일이 기존 작가의 저작권을 침해할 위험. → 스타일 프로파일 등록제 + 레퍼런스 업로드 동의 + DMCA 창구.
- **기술**: 외부 이미지 모델의 가격/정책 변화. → 모델 어댑터 추상화, 2–3개 벤더 병행.
- **품질**: 얼굴 정합이 일관되게 안 나옴. → 얼굴은 전용 파이프라인 + 인간 피니시 필수.
- **시장**: 직접 경쟁자 출현. → 후처리·검수·커뮤니티로 방어.
- **운영**: AI 비용이 매출보다 빠르게 증가. → 캐싱, 계층적 모델 사용, 배치 API.

## 10. 이 문서에 대한 열린 질문 (Open Questions)
