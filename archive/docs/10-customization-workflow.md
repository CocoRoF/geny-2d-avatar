# 10. 커스터마이징 워크플로우 (Customization Workflow)

> **한 줄 요약**: 사용자가 **"내 것" 이라고 느낄 때까지 다듬는 과정**을 설계한다. 프롬프트, 프리셋, 파츠별 조정, 버전 계보, 협업 리뷰, 분기(branch)까지 — 모든 "반복" 을 편하게 만드는 게 목표.

---

## 1. 원칙

1. **"생성" 은 시작이고, "커스터마이징" 이 본론이다.** 사용자는 1회차 결과에 만족하지 않는다.
2. **버전은 기본**이다. 되돌리기/분기/비교가 1급 기능.
3. **부분 재생성이 기본값**이다. 전체 재생성은 명시적 선택일 때만.
4. **기억을 떠넘기지 않는다**. 시스템이 시도한 것, 사용자가 마음에 들어한 것, 실패한 이유를 기록해 다시 묻지 않는다.
5. **협업을 전제로 설계**한다. 솔로도 쓸 수 있지만 팀이 들어오면 자연스럽게 확장.

---

## 2. 커스터마이징 계층 (Customization Layers)

```
 L1. Style Profile            ← 전체 룩 (팔레트, 선, 톤)
      ↓
 L2. Template & Presets       ← 리그/체형/기본 파츠 프리셋
      ↓
 L3. Part-level Prompt        ← 각 슬롯의 지시/레퍼런스
      ↓
 L4. Part-level Manual Edit   ← 앵커, 색 보정, 선 보정 수동
      ↓
 L5. Post-Processing Override ← 고급 사용자: 후처리 파라미터
```

위 레이어는 **상위가 하위를 덮어쓰는 구조**. 하위에서 수정한 것은 상위를 바꿔도 유지되도록 **"수동 수정 잠금(manual lock)"** 이 자동 적용된다.

---

## 3. 스타일 프로파일 (Style Profile)

### 3.1 정의

- 아바타 전체에 걸친 **룩(look)** 정의. 팔레트, 선 두께, 톤, 레퍼런스 이미지, 임베딩.
- 하나의 프로파일은 **여러 아바타**에 재사용 가능.

### 3.2 생성 경로

- **From scratch**: 템플릿 프리셋 복제 후 수정.
- **From upload**: 사용자가 업로드한 이미지들로 자동 추정 (CLIP + 색 팔레트 추출).
- **From selected avatar**: 이미 만든 아바타의 파츠에서 거꾸로 프로파일 추출.

### 3.3 잠금(Lock)

- Lock 이 걸린 프로파일은 **편집 불가**. 파생(fork)만 가능.
- 스튜디오/IP 운영자가 브랜드 통일성을 유지하는 용도.

### 3.4 권한

- 개인: 자기 프로파일 소유.
- 조직: 조직 공유 프로파일 (조직 내 멤버만).
- 퍼블릭: 마켓플레이스 공개.

---

## 4. 프리셋 (Preset)

### 4.1 프리셋이란

- **"이대로 만들면 60% 는 완성"** 되는 시작점.
- 구성: 템플릿 + 스타일 프로파일 + 파츠 프롬프트 세트 + 네거티브 예시.

### 4.2 공식 프리셋 예

- `Soft Pastel Shoujo` — 따뜻한 파스텔, 얇은 선, 큰 눈.
- `Lo-fi Korean Webtoon` — 진한 선, 쿨톤 팔레트.
- `Chibi Streamer` — 2등신, 과장된 표정.
- `Urban Realism` — 얇은 선, 대비 낮은 팔레트.
- `Game NPC — Kingdom Fantasy` — 전신, 갑옷/로브 친화.

### 4.3 커뮤니티 프리셋

- 사용자가 자신의 세팅을 프리셋으로 저장 후 공유. 태그/검색/별점.

---

## 5. 파츠별 프롬프트 & 레퍼런스

### 5.1 스코프

- 각 슬롯의 `prompt_scope` 안에서만 입력 반영.
- 슬롯에 맞지 않는 입력은 UI 가 부드럽게 리다이렉트(예: "이건 의상에 더 어울려요 → 의상 슬롯으로 이동?").

### 5.2 레퍼런스 이미지

- 슬롯당 최대 3장.
- "무게(strength)" 슬라이더(벤더 지원 시): 0–100%.
- 레퍼런스가 외부(저작권 미상) 이미지일 경우 **업로드 시 경고**.

### 5.3 프롬프트 템플릿 도우미

- 슬롯에 진입하면 **프롬프트 치트시트** 툴팁 제공.
- 예시: `hair_front` 에서 "스타일(웨이브/스트레이트/뱅), 색상, 길이" 필드를 구조화해 자유 입력과 병행.

---

## 6. 버전 관리 (Versioning)

### 6.1 버전의 단위

- **아바타 버전**: 사용자가 의미 있는 "저장" 할 때마다 증가 (자동 저장은 스냅샷만).
- **파츠 인스턴스**: 각 생성 시 새 ID. 히스토리 무한 보존(보관 한도는 플랜별).

### 6.2 히스토리 뷰

- 타임라인: 날짜/액션/썸네일.
- 파츠 필터: 이 파츠의 과거 버전만.
- **"이 버전으로 되돌리기"**: 전체 또는 파츠 단위.

### 6.3 스냅샷 vs 체크포인트

- 스냅샷: 자동, 편집 단위.
- 체크포인트: 사용자가 이름을 붙이는 수동 저장. 협업/발표용.

### 6.4 분기(Branch)

- 한 아바타에서 "의상 A 버전" / "의상 B 버전" 을 동시에 유지.
- 브랜치는 **체크포인트 기반**으로 출발.
- 브랜치 간 **비교/머지** 가능 (머지는 파츠 단위 체리픽).

---

## 7. 파츠 단위 편집 & 수동 조정

### 7.1 수동 조정의 종류

1. **앵커 수동 조정**: 자동 탐지 값 위에 드래그 오버라이드.
2. **색 조정**: 슬라이더(hue/sat/value) 로 파츠 단일 조정.
3. **선화 톤 조정**: 선 색/굵기 미세 변경.
4. **가시성 토글**: 특정 파츠 숨김/표시.
5. **Z-order 미세 조정**: 스펙 범위 내.

### 7.2 수동 수정 잠금

- 사용자가 수동 조정을 한 파츠는 **자동 재생성 대상에서 제외**.
- 명시적으로 해제해야 다시 AI 가 덮어쓸 수 있음.

### 7.3 오리지널 보존

- 모든 파츠는 **raw 버전** 과 **후처리/수동 편집 버전** 을 별도로 보관. 언제든 raw 로 돌아갈 수 있다.

---

## 8. 재생성 전략 (Regeneration Strategies)

### 8.1 세 가지 버튼

- **Variation**: 같은 프롬프트 + 다른 시드. 느낌이 마음에 들어서 "비슷하게 더".
- **Refine**: 현재 결과를 기반으로 "살짝 조정" (벤더의 edit 모드 활용).
- **Rebuild**: 처음부터 다시. 시드/프롬프트/레퍼런스 모두 재구성.

### 8.2 UI 제시

- 기본은 Refine. 반복 Refine 후 개선되지 않으면 Variation 제안.
- 3회 Rebuild 후에도 실패 → "인간 리뷰 권장" 배지.

### 8.3 "A/B 슬라이더"

- 현재 결과 vs 새 결과 중간 슬라이더. 사용자가 0–100% 로 선택.
- 선택된 파츠 버전이 채택됨 (블렌딩이 아닌 선택).

---

## 9. 협업 워크플로우

### 9.1 역할

- **Viewer**: 보기만.
- **Editor**: 편집 가능.
- **Reviewer**: 승인/반려 + 코멘트.
- **Admin**: 권한/빌링.

### 9.2 리뷰 절차 (Enterprise)

```
 Editor 편집 → "리뷰 요청" 클릭 → Reviewer 큐로 이동
  → Reviewer 가 파츠/포즈별 코멘트 → 승인/반려
  → 승인 시 자동으로 "프로덕션 채널" 로 pin
  → 반려 시 Editor 에게 돌아감, 코멘트 잔류
```

### 9.3 동시 편집

- 초기(β): 락 기반. 누군가 편집 중이면 다른 사람은 뷰어.
- GA: 파츠 단위 락. 섬세한 동시 편집은 지원하지 않음(복잡성 ↑).

### 9.4 코멘트

- 파츠별 / 포즈별 / 전체.
- 해결됨 토글, 스레드형.
- 알림: 앱 내 / 이메일.

## 10. 배치 생성 (Batch)

### 10.1 CSV/JSON 업로드

```csv
name,body,style_profile,prompt_face,prompt_hair,prompt_cloth
Alice,halfbody,soft_pastel,freckles,short_wavy_brown,school_uniform_blue
Bob,halfbody,soft_pastel,tanned,long_straight_black,hoodie_grey
...
```

### 10.2 대시보드

- 전체 진행률, 개별 상태, 실패 목록.
- 실패한 캐릭터만 재생성 버튼.
- 일괄 다운로드(ZIP) / 일괄 export.

### 10.3 우선순위

- 배치는 기본 저속 큐. 프리미엄 티어는 일반 속도.

---

## 11. 라이브러리 & 재사용

### 11.1 라이브러리 구성

- 내 것(My), 조직(Organization), 퍼블릭(Community).
- 각 항목에 태그/검색/미리보기.

### 11.2 파츠 재사용

- 라이브러리의 파츠를 에디터에 드래그 → 현재 아바타의 슬롯에 assign.
- 컬러 컨텍스트 자동 적응(옵션): 현재 아바타의 팔레트에 맞춰 자동 recolor.

### 11.3 모션 팩 재사용

- 모션 팩은 파라미터 이름이 템플릿과 일치할 때만 호환.
- UI 에서 "호환성 체크" 자동.

---

## 12. 학습 가속 (Onboarding Accelerators)

### 12.1 샘플 프로젝트

- 공식 샘플 3개: "첫 VTuber", "NPC 배치", "브랜드 마스코트".
- 각 샘플에 **단계별 투어**.

### 12.2 가이드 에이전트 (옵션)

- 사이드바 도우미: 현재 결과를 보고 "앞머리가 얼굴을 너무 덮은 것 같아요. 재생성 해볼까요?" 식의 **구체적 제안**.
- 거부감을 최소화하기 위해 기본은 off, 필요 시 on.

### 12.3 단축어 체계

- 숙련자용 키보드 중심 흐름 별도 제공.

---

## 13. 데이터 & 자산 거버넌스

### 13.1 소유권

- 기본: 자신이 생성한 자산은 자신이 소유.
- 조직 계정에서 만든 자산은 조직 소유.

### 13.2 삭제

- 사용자 자산 삭제는 30일 유예 후 영구 삭제.
- 조직 탈퇴 시 자산의 귀속 정책에 따라 이전/삭제.

### 13.3 감사 로그

- 생성/삭제/공유/export 모두 감사 로그.
- Enterprise 는 다운로드 가능.

---

## 14. 큐·비용·쿼터 UX

- 사용자가 자기 **월 쿼터 상태** 를 명확히 본다.
- 과금 전 **예상 비용** 을 보여준다 (예: "이 아바타 한 대 생성 ≈ $0.45, 남은 월 크레딧 12.30").
- 프리티어 사용자가 상한에 도달했을 때 **다운그레이드 경로**(저비용 모델로 계속).

---

## 15. 품질 피드백 루프

- 사용자에게 결과 만족도(1–5) 및 이유 태그를 요청(강요 금지, 무간섭 원칙).
- 수집된 신호는 **프리셋/프로파일 개선** 과 게이트 임계값 재조정에 사용.
- 개인 데이터 분리/집계 수준에서만 사용.

---

## 16. 사례별 워크플로우 예시

### 16.1 VTuber: "첫 대회 방송 3일 전"

- 저녁 20:00: 레퍼런스 업로드, 프리셋 선택, 생성.
- 20:15: 프리뷰, 앞머리/눈 수정 요청.
- 20:45: 브랜치 생성 ("라이트 버전"), 시즌 의상 실험.
- 21:10: VTube Studio 테스트, 립싱크 확인.
- 21:30: Export, 기록.

### 16.2 스튜디오: "시즌 3 등장 NPC 15명"

- 월요일: 스타일 프로파일 픽스, CSV 작성.
- 화요일 새벽 00:00 배치 큐.
- 화요일 09:00 결과 검토, 실패 3명 재생성.
- 목요일: QA 리뷰, 승인.
- 금요일: Unity 플러그인 임포트.

### 16.3 브랜드: "로고 컬러에 맞춘 마스코트"

- 프리-리그드 템플릿 선택.
- 컬러 팔레트 잠금.
- 로고 파츠 삽입(커스텀 슬롯).
- Web SDK 임베드 코드 복사.

---

## 17. 열린 질문

- **공동 편집 수준**: 풀 동시 편집(Figma 수준) 은 비용 대비 효용이 있는가? 초기 엔터프라이즈 피드백으로 결정.
- **AI 어시스트 톤**: 적극적 제안 vs 무간섭. 기본값 A/B.
- **브랜치 수 상한**: 플랜별 상한 존재해야 하는지.
