# 07. 자동 적합 시스템 (Auto-Fitting System)

> **한 줄 요약**: 후처리가 끝난 파츠를 **표준 리그 템플릿 위에 정확히 얹는** 공정. 앵커 탐지 → 리그 맞춤 → 디포머 주입 → 깨짐 복구까지를 책임진다. "움직일 때도 안 깨지는지" 를 결정하는 단계.

---

## 1. 문제 정의

### 1.1 왜 자동 적합이 필요한가

- AI 는 파츠를 **"비슷한 위치"** 에 생성한다. 완벽히 같은 좌표가 아니다.
- 리그 템플릿의 디포머는 **"정확한 좌표 기대값"** 을 갖는다.
- 어긋난 파츠를 그대로 디포머에 넣으면 **변형 시 튀거나 뚫린다**.
- 특히 얼굴 이목구비, 눈 위치, 머리 탑(crown) 이 조금만 어긋나도 "사람이 아닌 느낌" 이 난다.

### 1.2 해법의 구조

```
 Parts (post-processed)           Rig Template
        │                                 │
        ▼                                 ▼
 ┌────────────────┐              ┌────────────────┐
 │ Landmark/Anchor │              │ Expected anchor │
 │   Detection     │              │   positions     │
 └────────┬────────┘              └────────┬────────┘
          └────────────┬──────────────────┘
                       ▼
                 [ Alignment ]
                       │
                       ▼
            [ Mesh Injection ]        ← 메쉬 좌표에 파츠 UV 매핑
                       │
                       ▼
            [ Deformer Binding ]      ← 파라미터 변형이 잘 작동하는지
                       │
                       ▼
            [ Break Detection ]       ← 극단 변형 시 깨짐 탐지
                       │
                       ▼
                 Success / Retry
```

## 2. 앵커 탐지 (Anchor Detection)

### 2.1 앵커 타입별 방법

| 앵커 타입 | 탐지 방법 | 신뢰도 |
|---|---|---|
| `bbox_center` | 알파 > 0 영역의 axis-aligned bbox 중심 | ≥ 0.99 |
| `alpha_centroid` | Σ α·xy / Σ α | ≥ 0.95 |
| `eye_pupil_center_l/r` | 얼굴 랜드마크 모델 (MediaPipe FaceMesh + 커스텀) | 0.85–0.95 |
| `mouth_center` | 얼굴 랜드마크 | 0.85–0.95 |
| `nose_tip` | 얼굴 랜드마크 | 0.85–0.95 |
| `head_top_center` | 알파 상단 + 중앙 수직선 | 0.90 |
| `hair_parting_centroid` | 세그멘테이션 + 수직 에지 검출 | 0.70–0.85 |
| `neck_top_center` | `face_base` 하단과 접하는 알파 경계 | 0.85 |
| `shoulder_l/r` | 포즈 키포인트 모델 | 0.80 |

### 2.2 다중 탐지기 투표(Ensemble)

- 얼굴 이목구비 같은 중요 앵커는 **2개 이상의 탐지기**로 투표.
- 예: MediaPipe + 자체 경량 모델 → IoU/distance 로 일치 여부.
- 불일치 시 신뢰도 -0.2.

### 2.3 신뢰도 게이트

- 신뢰도 ≥ 0.85 → 자동 적합 계속.
- 0.7 ≤ 신뢰도 < 0.85 → 자동 적합 진행 + 사용자에게 **"수동 확인 권장"** 배지.
- 신뢰도 < 0.7 → 자동 적합 **중단**, 인간 리뷰 큐.

### 2.4 사용자 수동 앵커

- 에디터 UI 에서 사용자가 직접 앵커 점을 끌어 놓을 수 있다.
- 이 값은 탐지 결과를 덮어쓰며, `anchors.json` 에 `source: "manual"` 로 기록.

## 3. 얼라인먼트 (Alignment)

### 3.1 변환의 제한

- **이동 + 균일 스케일** 만 허용. 회전/비균일 스케일 금지.
- 이유: 회전은 선화의 픽셀 품질을 깎고, 비균일 스케일은 얼굴 비율을 왜곡.

### 3.2 알고리즘

1. 탐지된 앵커 위치 `p_obs`.
2. 기대 앵커 위치 `p_exp` (템플릿 스펙).
3. 단일 파츠 내 **하나의 앵커**: 이동만 (스케일 유지).
4. 단일 파츠 내 **두 개 이상의 앵커** (예: 얼굴의 눈/코/입): 최소제곱 이동 + 스케일 (Procrustes, 회전 제외).

### 3.3 제한/상한

- 평행 이동 상한: 파츠 UV 박스의 각 축 15%.
- 스케일 변동 상한: ±8%.
- 초과 시 → **재생성 요청** (alignment failure).

### 3.4 다중 파츠 공동 정렬(Joint Alignment)

얼굴 계열(face_base, eye_*, brows, nose, mouth)은 서로 상대적 위치가 중요하므로 **묶어서** 정렬한다.

- 공통 기준: `face_base` 의 `bbox_center` + 이목구비 앵커.
- 이목구비 파츠 자체는 얼굴 기준계에서 기대 위치에 정렬.

## 4. 메쉬 주입 (Mesh Injection)

### 4.1 기본 방향

- 리그 템플릿은 각 파츠마다 **미리 정의된 메쉬** 를 갖는다(03 참조).
- 자동 적합은 파츠 PNG 의 **UV 를 해당 메쉬에 매핑**한다.

### 4.2 매핑 방식

- 메쉬의 UV 좌표(0–1) 는 고정.
- 파츠 PNG 의 픽셀 좌표를 "UV 기대 박스" 에 맞춰 rescale.
- 결과: 메쉬 정점이 파츠 PNG 의 정확한 위치를 샘플링.

### 4.3 UV 외곽 여유(Padding)

- 파츠 PNG 에 **1–2 px 외곽 여유** 확보 (메쉬가 극단 변형 시 UV 가장자리 샘플링 문제 방지).
- 이 여유는 후처리 Stage 6 에서 보장.

---

## 5. 디포머 바인딩 검증 (Deformer Binding)

### 5.1 목적

- 메쉬가 디포머에 정상 연결되었는지, 파라미터 값 변화가 메쉬 정점에 제대로 전달되는지 확인.

### 5.2 방법

- 각 디포머의 **대표 파라미터** 를 최소/최대/중간 값으로 설정해 메쉬 정점 이동량 기록.
- 기대치와 비교:
  - 전혀 움직이지 않음 → **미바인딩 에러**.
  - 기대치 대비 ±ε 초과 → **바인딩 오류 에러**.

### 5.3 자가교차(Self-intersection) 체크

- 극단 변형 시 **메쉬 삼각형의 뒤집힘** 또는 **자가 교차** 가 발생하는지 감지.
- 발생 시 해당 변형 지점에서 파츠가 깨질 가능성 → **깨짐 탐지**(§7)로.

---

## 6. 얼굴 특수 처리 (Face Special Case)

얼굴은 자동 적합이 가장 어려운 영역이다. 별도 서브파이프라인을 둔다.

### 6.1 얼굴 전용 단계

```
 face_base 정렬
     │
 이목구비 랜드마크 탐지(다중 탐지기)
     │
 눈/코/입 파츠 위치 계산
     │
 파츠별 개별 정렬 (이동+스케일)
     │
 얼굴 기울기 탐지 (템플릿은 수직 얼굴 기대)
     │
 기울기 > 임계 → 사용자에게 재촬영 또는 재생성 안내
     │
 통과
```

### 6.2 얼굴 기울기 제약

- 원본 레퍼런스가 이미 기울어져 있으면 자동 적합이 어렵다.
- 기울기 > 8° → 사용자에게 "정면 일러스트 권장" 안내.
- 기울기 > 18° → 자동 적합 중단.

### 6.3 얼굴 보정 실패 시

- 얼굴은 자동 재생성보다 **인간 리뷰 + 수동 조정** 가중치를 높인다.
- 사용자에게 이목구비 앵커를 직접 찍는 UI 를 제공 (어시스트 포함).

## 7. 깨짐 탐지 (Break Detection)

자동 적합의 마지막 검문소. 변형을 시뮬레이션하며 "움직일 때 이상한가" 를 본다.

### 7.1 테스트 포즈 세트

템플릿 `test_poses/validation_set.json` 에 정의됨. 예:

```json
{
  "poses": [
    { "id": "neutral",        "params": {} },
    { "id": "blink_left",     "params": { "eye_open_l": 0 } },
    { "id": "blink_right",    "params": { "eye_open_r": 0 } },
    { "id": "mouth_open",     "params": { "mouth_open": 1 } },
    { "id": "head_turn_l30",  "params": { "head_angle_x": -30 } },
    { "id": "head_turn_r30",  "params": { "head_angle_x": 30 } },
    { "id": "head_tilt_l30",  "params": { "head_angle_z": -30 } },
    { "id": "head_nod",       "params": { "head_angle_y": 20 } },
    { "id": "smile",          "params": { "mouth_form": 1, "eye_smile_l": 1, "eye_smile_r": 1 } },
    { "id": "breath_in",      "params": { "body_breath": 1 } }
  ]
}
```

### 7.2 탐지 항목

- **메쉬 자가교차**: 삼각형 뒤집힘(winding 반전).
- **얼굴 뚫림**: 고개 회전 시 `hair_side_*` 뒤 `face_base` 의 빈 영역이 드러남.
- **알파 점프**: 인접 파츠 경계에서 알파 차 급변 (이음매 균열).
- **색 점프**: 경계에서 ΔE 급변.
- **파츠 튕김**: 파라미터 미세 변경에 파츠가 몇 px 이상 점프.

### 7.3 스코어링

각 포즈에 0–10 감점 부여, 합산 → 100 기반 점수.

### 7.4 실패 대응

- 점수 ≥ 78 → 자동 통과.
- 70 ≤ 점수 < 78 → 경미한 경고, 통과.
- 60 ≤ 점수 < 70 → 사용자에게 "수동 보정 추천" 배지, 인간 리뷰 큐 선택지 표시.
- 점수 < 60 → **깨진 파츠 명시** + 해당 파츠 재생성 트리거.

## 8. 구체적 케이스별 휴리스틱

### 8.1 머리카락 뚫림

- **증상**: 고개 좌/우 회전 시 얼굴 가장자리 뒤로 `hair_side_*` 가 밀려 `face_base` 가 비침.
- **원인**: `hair_side_*` 의 세로 높이가 얼굴 높이보다 짧음.
- **대응**: 해당 파츠 스펙의 `min_alpha_area_frac` 위반 여부 확인 → 재생성 시 "side hair length ≥ face height" 제약 추가.

### 8.2 앞머리 공중부양

- **증상**: 고개 끄덕임 시 앞머리가 얼굴과 함께 움직이지 않고 **한 박자 먼저** 움직임.
- **원인**: 앞머리 앵커가 얼굴과 분리되어 있지만 물리 응답이 과함.
- **대응**: 물리 프리셋을 `normal → light` 로 자동 조정 제안.

### 8.3 눈동자 흰자 이탈

- **증상**: `eye_ball_x/y` 극단값에서 홍채가 흰자를 벗어남.
- **원인**: `eye_iris` 파츠 크기가 흰자 대비 크거나, 제한 마스크 부재.
- **대응**: 흰자 마스크를 홍채에 AND 연산으로 클램프 (런타임 마스크). 근본 수정은 재생성.

### 8.4 입속 구강 미존재

- **증상**: `mouth_open=1` 에서 입속이 투명.
- **원인**: `mouth_inner` 선택 파츠가 비어 있음.
- **대응**: 기본 구강 텍스처 자동 주입 (색 context 에 맞춰).

### 8.5 목-얼굴 단차

- **증상**: `face_base` 와 `neck` 의 경계에 **색 단차** 발견.
- **원인**: 두 파츠가 서로 다른 벤더/시드로 생성된 후 색 통합이 경계에서 약함.
- **대응**: Stage 5 (Seam) 재실행 + 경계 페더 강화.

## 9. 실시간 프리뷰와의 관계

- 에디터의 프리뷰 플레이어(09 참조) 는 자동 적합의 결과를 그대로 렌더링.
- 깨짐 탐지에서 발견한 포즈는 **프리뷰 플레이어에서 재현 버튼** 으로 즉시 재생 — 사용자가 문제를 눈으로 확인 가능.

## 11. 내부 API (개요)

```python
class AutoFitPipeline:
    async def run(self, avatar_id: str, opts: FitOptions) -> FitResult: ...

class FitOptions(BaseModel):
    strictness: Literal["loose", "normal", "strict"] = "normal"
    allow_manual_anchor_override: bool = True
    run_break_detection: bool = True

class FitResult(BaseModel):
    status: Literal["auto_pass", "needs_review", "failed"]
    score: int
    failures: list[FitFailure]
    updated_artifacts: list[str]  # 업데이트된 파일 경로/ID
```

---

## 12. 품질 개선 전략 (Improvement Loop)

- 자동 적합 실패 사례를 **익명화 샘플**로 모아, 앵커 탐지 모델의 약점을 주기적으로 보강.
- "실패 상위 10 케이스" 대시보드 → 스프린트별 공격 대상 선정.

---

## 13. 열린 질문

- 얼굴 랜드마크 모델을 내부로 끌어올지, 벤더(MediaPipe 등) 에 의존할지.
- 브레이크 감지 시 **자동 보정(mesh re-bind)** 을 시도할지, 항상 재생성을 요구할지 (복잡성 vs 품질).
- 사용자 수동 앵커 편집을 PRO 기능으로만 개방할지.
