# 14. 로드맵과 마일스톤 (Roadmap & Milestones)

> **한 줄 요약**: MVP → α → β → GA → Post-GA 까지 **"지금 이 단계를 끝냈다고 어떻게 아는가"** 를 지표로 명확히 못 박는다. 범위를 줄이고 품질 게이트를 높이는 방향이 원칙.

---

## 1. 기본 규칙

1. **Ship a slice, not a skeleton.** 각 단계는 end-to-end 사용자 가치(생성→미리보기→export) 를 관통해야 한다.
2. **Done 은 지표로 판단한다.** "대충 완성" 은 없다.
3. **범위 변화는 문서화**: 범위를 바꿀 때 [index.md](./index.md) CHANGELOG 와 이 문서의 해당 섹션을 업데이트.
4. **MVP 는 "팔기 위한 것이 아니라 배우기 위한 것"**. 유료화 시점은 β 이후.

---

## 2. 단계 요약표

| 단계 | 기간 (목표) | 범위 | 완료 기준 |

| **Internal MVP** | 2026 Q2 말 | 내부 5명이 아바타 만들어 VTS 띄우기 | 내부 아바타 25대 성공, 평균 검수 점수 ≥ 70 |

## 3. Foundation (2026 Q2 초)

### 3.1 목표

- 팀이 **"실제 아바타 한 대"** 를 end-to-end로 돌릴 수 있는 인프라·뼈대 구축.

### 3.2 워크스트림

- **Infra**: K8s 클러스터, CI/CD, 관측성 스택, 시크릿 관리.
- **Data**: Postgres/S3/Redis, 스키마 초판([12 참조](./12-data-schema-and-api.md)).
- **Rig**: 공식 템플릿 1종 (`tpl.base.v1.halfbody`) 손 리깅으로 완성.
- **Pipeline**: 단일 아바타 DAG 구현(§[02 §4](./02-system-architecture.md)). 실제 AI 호출은 nano-banana 어댑터 1개.
- **Frontend**: 에디터 기본 레이아웃 + 프리뷰 플레이어 기초.

### 3.3 완료 기준 (Exit)

- [ ] 단일 아바타를 API 호출로 생성 → 프리뷰 → Cubism export 까지 수동 테스트 성공.
- [ ] CI 에서 골든 1 아바타 회귀 검사 자동.
- [ ] 관측 대시보드 3종 기본 동작.
- [ ] 개발자 온보딩 1일.

---

## 4. Internal MVP (2026 Q2 말)

### 4.1 목표

- 내부 아티스트 5명이 **"자기 캐릭터로 VTuber 가능한 아바타"** 를 만들 수 있다.
- 페인 포인트 목록화, 외부 공개 전 빌딩 로드맵 우선순위 재조정.

### 4.2 주요 기능

- Editor 사이드바 + 파츠별 재생성.
- 스타일 프로파일 기본 구현.
- 자동 앵커 탐지 + 수동 보정 UI.
- 검수 렌더러 v0 (포즈 10종, 점수 산출).
- Export: Cubism + VTube Studio.

### 4.3 완료 기준

- [ ] 내부 5명이 각자 1대 이상 생성, 총 25대 이상.
- [ ] 평균 자동 검수 점수 ≥ 70.
- [ ] 평균 TTFE ≤ 2시간.
- [ ] 자주 발생 이슈 top-5 가 문서화되고, α 이전 처리 계획 존재.

### 4.4 미포함 (Descoped)

- 결제/빌링.
- 마켓플레이스.
- Unity/Unreal/Mobile.
- 전신 템플릿.
- 배치 생성.

---

## 5. Closed Alpha (2026 Q3)

### 5.1 목표

- 30명 외부 크리에이터 초청.
- 유료화 전 **품질 게이트 상향**, 대규모 사용자의 현실 워크로드 검증.

### 5.2 신규 기능

- Full body 템플릿 추가.
- Style Profile 공유 (organization 수준).
- 모션 팩 선택 UI.
- Web SDK v0 공개 (embed 한 줄).
- Validation report 의 "사람 말 문구" 강화.
- 에디터 브랜치 기본 기능.

### 5.3 완료 기준

- [ ] **NSM (WSAC)** 주간 ≥ 40.
- [ ] NPS ≥ 30.
- [ ] 완주율 ≥ 40%.
- [ ] 자동 검수 점수 평균 ≥ 75.
- [ ] 심각(P1) 버그 0건 7일 연속.

### 5.4 대표 리스크

- AI 벤더 비용 급증(사용량 ↑). → 라우팅/캐시 강화.
- 사용자 기대 오버슈팅. → "반자동" 마케팅 톤 유지.

---

## 6. Open Beta (2026 Q4)

### 6.1 목표

- **공개 가입 + 유료 플랜 런칭**.
- 지속 가능성(경제성·품질·신뢰) 1차 증명.

### 6.2 신규 기능

- 결제/쿼터/크레딧.
- 무료/Pro/Team 플랜.
- 배치 생성 (CSV 업로드).
- Unity SDK β.
- Chibi/Feline 템플릿.
- 커뮤니티 프리셋 베타.
- 사용자 수동 앵커 편집 UI.

### 6.3 완료 기준

- [ ] WSAC 주간 ≥ 2,000.
- [ ] 유료 전환 ≥ 4%.
- [ ] 아바타당 평균 AI 비용 ≤ $0.60.
- [ ] 가용성 ≥ 99.5%.
- [ ] 지원 티켓 / 1k 유저 ≤ 30/월.
- [ ] 초기 6개월 이내 Churn ≤ 10% / 월.

### 6.4 대표 리스크

- 트래픽 급증으로 큐 폭주. → 오토스케일+우선순위 버킷.
- 저작권 이슈(업로드 레퍼런스). → 정책·검증 강화.

---

## 7. GA (2027 Q1)

### 7.1 목표

- **엔터프라이즈 급 신뢰**, 커뮤니티 에코시스템 시동.

### 7.2 신규 기능

- 엔터프라이즈 플랜: SSO, 감사 로그 export, 전용 쿼터.
- 마켓플레이스 (프리셋·파츠·모션팩 유료/무료 공유).
- Unreal β.
- Mobile SDK β.
- Webhooks 공식.
- 플러그인 SDK 초판.
- IP 보호 기능(스타일 등록/침해 신고).

### 7.3 완료 기준

- [ ] 엔터프라이즈 계약 ≥ 3 체결.
- [ ] 가용성 ≥ 99.9%.
- [ ] 글로벌 리전 2개.
- [ ] 프리미엄 QA 리뷰 워크플로우 문서화.
- [ ] 마켓플레이스 MAU ≥ 5k.

### 7.4 대표 리스크

- 경쟁 제품 출시. → 후처리/검수/커뮤니티 해자 강화.
- 엔터 계약 요구사항 폭증. → 계약 전 기능 요구사항 표준화.

---

## 8. Post-GA (2027 Q2–Q3)

### 8.1 목표

- **시장 지배력 + 플랫폼화**.

### 8.2 기능

- 플러그인 오픈: 후처리/어댑터/검수기 3rd party.
- 자체 파인튜닝 모델(스타일별 LoRA) 공개(유료).
- 3D 호환 탐색(별도 제품 분리 가능).
- 모바일 앱(보기/공유).
- 커뮤니티 수익 분배.

### 8.3 완료 기준

- [ ] WSAC 20k+.
- [ ] 월 매출 ≥ $250k.
- [ ] 3rd party 플러그인 ≥ 20.
- [ ] 이탈률 ≤ 6%/월.

## 11. 외부 의존 이벤트(Risk-driven)

- **Cubism SDK 정책 변경** → Export/ Web SDK 영향. 분기마다 공지 모니터.
- **Google nano-banana 정책/가격 변경** → 어댑터 비중 조정.
- **규제(생성형 AI) 변경** → 콘텐츠 정책 업데이트.

## 14. 예산 / 현금 흐름 (요약)

## 15. "안 하기로 한 것" 기록 (Explicit No-list)

단계별로 **안 하는 것**을 기록해 우선순위를 지킨다.

- MVP 동안: 빌링/마켓/3D/Unreal/모바일.
- α 동안: 결제/대규모 콜라보/플러그인.
- β 동안: 완전 자동화 UX.
- GA 전: 3D.
