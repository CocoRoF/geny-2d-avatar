## 공통 원칙

1. **탐지(Detect) → 분류(Triage) → 완화(Mitigate) → 소통(Communicate) → 회고(Postmortem)** 5 단계를 순서대로 밟는다. 단계 건너뛰기 금지 — 건너뛰면 회고에 근거가 사라진다.
2. **롤백은 "먼저 복구, 나중에 분석"**. 증거 수집을 이유로 장애를 연장하지 않는다 (`docs/02 §10` 재시도/멱등 정책 덕분에 롤백 안전).
3. **소통 템플릿 준수**. 첫 공지에 (a) 영향 범위 (b) 시각 (c) 다음 업데이트 시각 3가지는 반드시 포함.
4. **회고는 비난 금지(blameless)**. 인시던트 24h 내 draft, 1주일 내 확정해 `progress/postmortems/YYYY-MM-DD-<slug>.md` 에 영속.
